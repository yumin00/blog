## 시퀀스 다이어그램
- 특정 행동이 어떤 객체와 어떻게 상호작용하는지 표현한 행위 다이어그램
- 시스템이 어떤 시나리오로 움직이는지 확인 가능
- API 등의 유즈케이스를 디테일하게 알 수 있음

### LifeLine
- 네모와 점선으로 이루어져 있음
- 네모 : 객체
- 점선 : 시간의 경과

### Activation
- 직사각형 막대
- Lifeline의 인스턴스가 다른 인스턴스와 상호작하며 활성화되어 있는 것을 나타냄

### Message
- 인스턴스 간 주고받은 메세지
- HTTP 통신 기준 요청과 응답으로 나타냄

| 유형      | 내용                      | 표현 방식                  |
|---------|-------------------------|------------------------|
| 동기 메세지  | 요청 후 반환이 올 때까지 대기       | 실선과 꽉 찬 화살표            |
    | 비동기 메세지 | 요청 후 반환을 기다리지 않고 다른 작업  | 실선과 선으로 이뤄진 화살표        |
    | 자체 메세지  | 자기 자신에게 메세지를 보냄         | 본인의 lifeline을 회기하는 화살표 |
    | 반환 메세지  | 요청에 대한 메세지 반환           | 점선과 선으로 이뤄진 화살표        |

### 흐름 제어
- if, for, while 과 같은 흐름을 제어하는 표현식

#### Guard
- 메세지에 대한 조건을 명시하는 방법
- 대괄호[] 안에 조건 명시
- ex) [price > 30,000] 배송비 무료
- 조건문을 만족하지 않으면 메세지를 호출되지 않음

#### Sequence Fragment
- 범위로 조건을 명시
- alternative(alt)
  - 조건에 따라 선택 사항이 여러개일 때
  - ex) [price > 30,000] 배송비 무료 / 아니면 배송비 유료
- option(opt)
  - 조건에 따라 선택 사항이 단 한 개일 경우
  - 처리하는 내용이 하나라면 Guard와 같음
  - ex) [price > 30,000] 배송비 무료 / 3,000원 할인
- loop
  - for, while 반복문
- parallel(par)
  - 병렬 처리
  - ex) 주문이 들어왔을 때 > 상품 확인 / 배송 여부 확인
